## 适用场景

从 HTML 或日志里抠 `<tag>...</tag>` 中间最短一段，若写 `.*` 默认**贪婪**，会从第一个 `<` 一直匹配到最后一个 `>`，中间全被吃掉。把量词改成 `*?` 或 `+?` 可变为**非贪婪**，每次尽量少匹配。

## 核心思路

`*` 匹配零次以上；`*?` 在整体仍能匹配成功的前提下，优先取最短。典型模式：`<div>(.*?)</div>` 配合 `re.DOTALL` 可跨行取第一个闭合块。

## 最小示例

```python
import re

html = "<p>第一段</p><p>第二段</p>"
greedy = re.findall(r"<p>.*</p>", html)      # 可能整串匹配
lazy = re.findall(r"<p>.*?</p>", html)      # 两个短匹配
print(lazy)
```

输出为 `['<p>第一段</p>', '<p>第二段</p>']`（在默认单行模式下本例已能区分；跨行时记得 `re.S`）。

## 常见坑

- 非贪婪仍可能匹配到**错误**的结束标签，HTML 应用解析器（如 `html.parser`、BeautifulSoup）更稳。  
- 嵌套括号时正则很难完全正确，复杂结构用语法分析或专用库。  
- 性能：极长字符串上来回回溯可能慢，必要时优化或限制输入长度。
