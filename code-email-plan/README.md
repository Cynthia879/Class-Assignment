# 代码邮箱计划 — 分组任务说明

## 目标

每个小组向全班**分享 1 条**在课程学习、实验或项目中**真实用到并掌握**的编程技巧，形成可检索的「技巧库」，类似往公共邮箱里投递一封「代码小贴士」。本班级计划为 **4 组**，认领与填表见 [`GROUPS.md`](GROUPS.md)。

## 规则

1. **产出数量**：每组 **恰好 1 篇**技巧文档（若多写可作为附录，但评优以主篇为准）。
2. **内容要求**：
   - 有**明确场景**（在什么情况下会遇到这个问题）。
   - 有**可运行或可复现**的简短示例（代码片段 + 说明）。
   - 注明**语言/环境**（如 Python 3.11、Node 20、Git 版本等）。
3. **提交方式**：
   - 在 [`GROUPS.md`](GROUPS.md) 中认领小组编号与主题一行。
   - 在 [`tips/`](tips/) 下新增 `groupXX-主题简述.md`（XX 为两位数组号），按 [`tips/_TEMPLATE.md`](tips/_TEMPLATE.md) 填写。
   - 通过 **Pull Request** 提交（推荐）或由仓库管理员代为合并。
4. **截止时间**：由班长在 `GROUPS.md` 顶部填写。

## 评分/展示建议（供班长参考）

- 技巧是否对多数同学有迁移价值。
- 示例是否简洁、可复现。
- 文档是否结构清晰、无抄袭（引用需注明出处）。
