# 代码邮箱计划 — 分组任务说明

## 目标

每个小组向全班**分享 1 条**在课程学习、实验或项目中**真实用到并掌握**的编程技巧，形成可检索的「技巧库」，类似向公共邮箱投递一封「代码小贴士」。当前认领表为 **4 组**，见 [`GROUPS.md`](GROUPS.md)；若组数变化，可直接编辑该表增删行。

## 规则

1. **产出数量**：每组 **恰好 1 篇**技巧文档（若多写可作为附录，课堂展示以主篇为准）。  
2. **内容要求**：  
   - 有**明确场景**（在什么情况下会遇到这个问题）。  
   - 有**可运行或可复现**的简短示例（代码片段 + 说明）。  
   - 注明**语言/环境**（如 Python 3.11、Node 20、Git 版本等）。  
3. **提交方式**：  
   - 在 [`GROUPS.md`](GROUPS.md) 中认领小组编号并填写主题一行。  
   - 在 [`tips/`](tips/) 下新增 `groupXX-主题简述.md`（XX 为两位数组号），按 [`tips/_TEMPLATE.md`](tips/_TEMPLATE.md) 填写。  
   - 通过 **Pull Request** 提交（推荐），或由仓库管理员代为合并。  
4. **截止时间**：在 `GROUPS.md` 顶部由课程组或仓库管理员填写；无统一要求时可写「无」或删除该行说明。

## 展示与互评参考

- 技巧是否对多数同学有迁移价值。  
- 示例是否简洁、可复现。  
- 文档是否结构清晰；引用他人内容须注明出处。
